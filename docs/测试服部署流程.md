# 测试服部署流程

本文档描述如何在 Linux 服务器上部署番茄小说爬虫测试服。

---

## 一、部署方式选择

### 方式一：Docker Compose 部署（推荐）

**适用场景**：
- 快速搭建测试环境
- 希望容器化部署
- 需要同时部署 MySQL 和 Redis

**优点**：
- 一键部署所有服务
- 环境隔离，避免依赖冲突
- 易于迁移和备份

### 方式二：手动部署

**适用场景**：
- 服务器已有 MySQL 和 Redis
- 需要更细粒度的控制
- Docker 不可用的环境

**优点**：
- 更灵活的资源配置
- 可以直接调试 Python 代码
- 无需 Docker 开销

---

## 二、方式一：Docker Compose 部署

### 2.1 前置要求

| 软件 | 版本 | 安装命令 |
|------|------|----------|
| Docker | 20.10+ | `curl -fsSL https://get.docker.com \| sh` |
| Docker Compose | 2.0+ | `apt install docker-compose-plugin` |

### 2.2 部署步骤

#### 步骤 1：克隆项目

```bash
cd /opt
git clone <your-repo-url> feature-crawler
cd feature-crawler
```

#### 步骤 2：配置环境变量

```bash
# 复制测试环境配置模板
cp .env.test .env

# 编辑配置
nano .env
```

**必需配置项**：
```bash
# 应用环境（测试服）
APP_ENV=test

# MySQL 配置（如使用外部数据库）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_test_password
MYSQL_DATABASE=fanqie_test

# Redis 配置（如使用外部 Redis）
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=2
```

#### 步骤 3：启动服务

```bash
# 仅启动爬虫服务（使用外部 MySQL/Redis）
docker compose --profile with-services up -d

# 启动所有服务（包括 MySQL 和 Redis）
docker compose --profile with-mysql up -d
```

#### 步骤 4：验证部署

```bash
# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f crawler

# 测试 API
curl http://localhost:8000/health
```

### 2.3 日常运维

```bash
# 重启服务
docker compose restart

# 停止服务
docker compose down

# 查看实时日志
docker compose logs -f crawler

# 进入容器调试
docker compose exec crawler bash

# 手动执行爬取任务
docker compose exec crawler python -m cli.main crawl --all --limit 30
```

### 2.4 数据持久化

Docker Compose 自动挂载以下目录：
- `./data` - 爬取的数据文件
- `./logs` - 日志文件
- `mysql_data` 卷 - MySQL 数据（如使用内置 MySQL）
- `redis_data` 卷 - Redis 数据

## 三、方式二：手动部署

### 3.1 前置要求

| 软件 | 版本 | 安装命令 |
|------|------|----------|
| Python | 3.10+ | `apt install python3.11 python3.11-venv` |
| MySQL | 8.0+ | `apt install mysql-server` |
| Redis | 6.0+ | `apt install redis-server` |
| uv（可选） | 最新 | `pip install uv` |

### 3.2 部署步骤

#### 步骤 1：安装系统依赖

```bash
# 更新软件源
sudo apt update

# 安装 Python 和开发工具
sudo apt install -y python3.11 python3.11-venv python3-pip

# 安装 MySQL 客户端库
sudo apt install -y default-libmysqlclient-dev pkg-config

# 安装系统字体（Playwright 依赖）
sudo apt install -y libnss3 libnspr4 libatk1.0-0 libatk-bridge2.0-0 libcups2 \
    libdrm2 libdbus-1-3 libxkbcommon0 libxcomposite1 libxdamage1 libxfixes3 \
    libxrandr2 libgbm1 libasound2 libpango-1.0-0 libcairo2
```

#### 步骤 2：创建应用用户（可选但推荐）

```bash
sudo useradd -r -s /bin/false fanqie
sudo mkdir -p /opt/fanqie-crawler
sudo chown -R fanqie:fanqie /opt/fanqie-crawler
```

#### 步骤 3：克隆项目

```bash
cd /opt
git clone <your-repo-url> feature-crawler
cd feature-crawler

# 如果是测试服，切换到测试分支（如果有）
# git checkout test
```

#### 步骤 4：安装 Python 依赖

**使用 uv（推荐）**：
```bash
# 安装 uv
pip3 install uv

# 创建虚拟环境并安装依赖
uv venv --python 3.11
source .venv/bin/activate
uv pip install -e .

# 安装 Playwright 浏览器
playwright install chromium
```

**使用 pip**：
```bash
# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -e .

# 安装 Playwright 浏览器
playwright install chromium
```

#### 步骤 5：配置环境变量

```bash
# 复制测试环境配置
cp .env.test .env

# 编辑配置
nano .env
```

**测试服配置示例**：
```bash
APP_ENV=test

# 数据库配置（localhost 表示同服务器）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_secure_password
MYSQL_DATABASE=fanqie_test

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=2

# 爬虫配置（测试服建议开启无头模式）
SPIDER_HEADLESS=true
SPIDER_DELAY_MIN=1
SPIDER_DELAY_MAX=3

# 日志配置
LOG_LEVEL=INFO
```

#### 步骤 6：初始化数据库

```bash
source .venv/bin/activate

# 方式 1：通过统计命令触发建表
python -m cli.main stats

# 方式 2：使用初始化脚本（如果有）
python scripts/init_db.py
```

#### 步骤 7：配置 systemd 服务（推荐）

创建服务文件：
```bash
sudo nano /etc/systemd/system/fanqie-crawler.service
```

添加以下内容：
```ini
[Unit]
Description=Fanqie Novel Crawler - Web API
After=network.target mysql.service redis.service

[Service]
Type=simple
User=fanqie
Group=fanqie
WorkingDirectory=/opt/fanqie-crawler
Environment="PATH=/opt/fanqie-crawler/.venv/bin"
ExecStart=/opt/fanqie-crawler/.venv/bin/python -m src.novel_crawler.main
Restart=on-failure
RestartSec=10

# 资源限制
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable fanqie-crawler
sudo systemctl start fanqie-crawler
sudo systemctl status fanqie-crawler
```

#### 步骤 8：配置定时任务（可选）

如果需要使用定时任务自动爬取：

```bash
# 编辑 crontab
crontab -e
```

添加以下内容（每天凌晨 2 点执行）：
```bash
# 每天凌晨 2 点执行爬取任务
0 2 * * * cd /opt/fanqie-crawler && .venv/bin/python -m cli.scheduler --once >> logs/cron.log 2>&1
```

或者使用 systemd timer（更推荐）：
```bash
sudo nano /etc/systemd/system/fanqie-crawler-daily.timer
```

```ini
[Unit]
Description=Daily Crawler Timer

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

```bash
sudo nano /etc/systemd/system/fanqie-crawler-daily.service
```

```ini
[Unit]
Description=Daily Crawler Job
After=network.target

[Service]
Type=oneshot
User=fanqie
WorkingDirectory=/opt/fanqie-crawler
Environment="PATH=/opt/fanqie-crawler/.venv/bin"
ExecStart=/opt/fanqie-crawler/.venv/bin/python -m cli.scheduler --once
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable fanqie-crawler-daily.timer
sudo systemctl start fanqie-crawler-daily.timer
```

## 四、验证部署

### 4.1 健康检查

```bash
# API 健康检查
curl http://localhost:8000/health

# 预期输出：{"status": "healthy", "timestamp": "..."}
```

### 4.2 检查服务状态

```bash
# systemd 方式
systemctl status fanqie-crawler

# Docker 方式
docker compose ps
```

### 4.3 执行测试爬取

```bash
# 爬取第一个分类的前 5 本书进行测试
python -m cli.main crawl --idx 0 --limit 5

# 检查爬取结果
python -m cli.main stats
```

### 4.4 检查日志

```bash
# 查看最新日志
tail -f logs/crawler-*.log

# 查看错误日志
grep ERROR logs/crawler-*.log | tail -20
```

## 五、常见问题

### Q1: Playwright 浏览器安装失败

```bash
# 安装系统依赖
playwright install-deps chromium

# 如果仍然失败，手动安装
apt install libnss3 libnspr4 libatk1.0-0 libatk-bridge2.0-0 libcups2 \
    libdrm2 libdbus-1-3 libxkbcommon0 libxcomposite1 libxdamage1 libxfixes3 \
    libxrandr2 libgbm1 libasound2 libpango-1.0-0 libcairo2
```

### Q2: MySQL 连接失败

```bash
# 检查 MySQL 服务
sudo systemctl status mysql

# 检查端口
netstat -tlnp | grep 3306

# 测试连接
mysql -h localhost -u root -p
```

### Q3: Redis 连接失败

```bash
# 检查 Redis 服务
sudo systemctl status redis

# 检查端口
netstat -tlnp | grep 6379

# 测试连接
redis-cli ping
```

### Q4: 权限问题

```bash
# 修复目录权限
sudo chown -R fanqie:fanqie /opt/fanqie-crawler
sudo chmod -R 755 /opt/fanqie-crawler/logs
sudo chmod -R 755 /opt/fanqie-crawler/data
```
