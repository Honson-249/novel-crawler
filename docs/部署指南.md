# 部署指南

> **注意**：测试服部署请参考 [测试服部署流程.md](测试服部署流程.md)，本文档主要介绍本地和通用部署。

## 环境准备

### 系统要求

- **操作系统**: Linux (Ubuntu 20.04+) / Windows Server 2019+ / macOS
- **Python**: 3.10 或更高版本
- **Redis**: 6.0 或更高版本
- **MySQL**: 8.0 或更高版本

### 依赖服务安装

#### 1. 安装 Redis

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install redis-server
sudo systemctl start redis
sudo systemctl enable redis
```

**验证安装:**
```bash
redis-cli ping
# 应返回：PONG
```

**Windows (使用 Chocolatey):**
```bash
choco install redis-64
```

#### 2. 安装 MySQL

**Ubuntu/Debian:**
```bash
sudo apt install mysql-server
sudo mysql_secure_installation
```

**创建数据库和用户:**
```sql
CREATE DATABASE novel_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'crawler'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON novel_db.* TO 'crawler'@'%';
FLUSH PRIVILEGES;
```

#### 3. 安装 Python 依赖

**推荐使用 uv:**
```bash
# 安装 uv
pip install uv

# 同步依赖
uv sync

# 安装浏览器
uv run playwright install chromium
```

**或使用 pip:**
```bash
python3 -m venv .venv
source .venv/bin/activate  # Linux/macOS
.venv\Scripts\activate     # Windows
pip install -e ".[dev]"
playwright install chromium
```

## 部署方式

### 方式一：systemd 服务（推荐，Linux）

#### 1. 创建服务文件

```bash
sudo nano /etc/systemd/system/fanqie-crawler.service
```

#### 2. 添加以下内容

```ini
[Unit]
Description=Fanqie Novel Crawler
After=network.target mysql.service redis.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/fanqie-crawler
Environment="PATH=/opt/fanqie-crawler/.venv/bin"
ExecStart=/opt/fanqie-crawler/.venv/bin/python -m cli.main crawl --all --detail
Restart=on-failure
RestartSec=300

[Install]
WantedBy=multi-user.target
```

#### 3. 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable fanqie-crawler
sudo systemctl start fanqie-crawler
sudo systemctl status fanqie-crawler
```

### 方式二：Docker 部署

```bash
# 构建镜像
docker build -t fanqie-crawler .

# 运行容器（使用外部数据库）
docker run -d \
  --name fanqie-crawler \
  -p 8000:8000 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/logs:/app/logs \
  --env-file .env \
  --add-host host.docker.internal:host-gateway \
  fanqie-crawler

# 查看日志
docker logs -f fanqie-crawler

# 停止容器
docker stop fanqie-crawler
```

### 方式三：Docker Compose（推荐用于测试服）

**使用外部数据库**：
```bash
# 仅启动爬虫服务
docker compose --profile with-services up -d
```

**使用内置 MySQL 和 Redis**：
```bash
# 启动所有服务（包括 MySQL 和 Redis）
docker compose --profile with-mysql up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f crawler

# 停止服务
docker compose down
```

**配置文件说明**：
- `docker-compose.yml` - 主配置文件
- `.env` - 环境变量配置

**环境变量配置示例**（测试服）：
```bash
# 复制测试环境配置
cp .env.test .env

# 编辑配置
nano .env
```

**必需配置项**：
```bash
APP_ENV=test

# MySQL 配置（如使用外部数据库）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=fanqie_test

# Redis 配置（如使用外部 Redis）
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=2
```

**注意事项**：
1. 容器内访问宿主机服务需要使用 `host.docker.internal` 或 `--network host`
2. 数据持久化：日志和数据目录会自动挂载到 `./logs` 和 `./data`
3. 内置 MySQL/Redis 的数据存储在 Docker volume 中

### 方式四：定时任务（crontab）

#### 1. 编辑 crontab

```bash
crontab -e
```

#### 2. 添加定时任务

```bash
# 每天凌晨 2 点爬取
0 2 * * * cd /opt/fanqie-crawler && .venv/bin/python -m cli.main crawl --all --detail >> logs/cron.log 2>&1

# 每 6 小时补充爬取缺失详情
0 */6 * * * cd /opt/fanqie-crawler && .venv/bin/python -m cli.main refill --limit 50 >> logs/cron.log 2>&1
```

## 日志管理

### 日志位置

- 爬虫日志：`logs/crawler-YYYY-MM-DD.log`
- 调度器日志：`logs/scheduler.log`
- 定时任务日志：`logs/cron.log`
- API 日志：`logs/api.log`

### 日志轮转

日志自动按天切转，保留 7 天。可修改 `src/novel_crawler/observability/logging_config.py` 中的配置。

## 监控告警

### 1. 检查爬取状态

```bash
# 查看最新批次数据量
python cli/main.py stats

# 验证数据完整性
python cli/main.py verify
```

### 2. 设置监控脚本

创建 `monitor.sh`:

```bash
#!/bin/bash

cd /opt/fanqie-crawler
source .venv/bin/activate

# 检查最新批次数据量
count=$(python -c "from src.novel_crawler.pipeline.storage import db_manager; from datetime import datetime; print(db_manager().get_batch_stats(datetime.now().strftime('%Y-%m-%d'))['total'])")

if [ "$count" -lt 1000 ]; then
    echo "警告：今日爬取数据量不足 (当前：$count)"
    # 发送告警通知（如：邮件、钉钉、企业微信）
fi
```

### 3. 添加到 crontab

```bash
# 每小时检查一次
0 * * * * /opt/fanqie-crawler/monitor.sh
```

## 故障排查

### 1. Redis 连接失败

```bash
# 检查 Redis 服务
sudo systemctl status redis

# 检查端口
netstat -tlnp | grep 6379

# 查看 Redis 日志
sudo tail -f /var/log/redis/redis-server.log
```

### 2. MySQL 连接失败

```bash
# 检查 MySQL 服务
sudo systemctl status mysql

# 检查端口
netstat -tlnp | grep 3306

# 测试连接
mysql -h 127.0.0.1 -u crawler -p
```

### 3. Playwright 浏览器问题

```bash
# 重新安装浏览器
playwright install chromium

# 安装系统依赖（Linux）
playwright install-deps chromium
```

### 4. 字体解密失败

字体映射会自动更新，如遇解密失败：

```bash
# 清除缓存
python -m cli.main clear-cache

# 重新初始化
python -m cli.main init-cache
```

### 5. 查看应用日志

```bash
# 查看最新日志
tail -f logs/crawler-*.log

# 查看错误日志
grep ERROR logs/crawler-*.log | tail -20
```

## 性能优化

### 1. 数据库优化

```sql
-- 添加索引
ALTER TABLE fanqie_ranks ADD INDEX idx_batch_date (batch_date);
ALTER TABLE fanqie_ranks ADD INDEX idx_book_id (book_id);

-- 定期清理旧数据
DELETE FROM fanqie_ranks WHERE batch_date < DATE_SUB(CURDATE(), INTERVAL 30 DAY);
```

### 2. Redis 优化

```bash
# 设置最大内存
redis-cli CONFIG SET maxmemory 512mb
redis-cli CONFIG SET maxmemory-policy allkeys-lru
```

### 3. 并发优化

如需支持高并发爬取，可修改 `src/novel_crawler/spiders/fanqie/config.py` 中的并发参数。

## 常见问题

### Q: 爬取速度慢？

A: 检查网络连接，调整爬虫延迟参数（`src/novel_crawler/spiders/fanqie/config.py`），确保未被封禁。

### Q: 数据不完整？

A: 运行 `python -m cli.main refill --limit 100` 补充爬取。

### Q: 如何清空数据重新爬取？

A:
```bash
# 清空指定日期数据
mysql -u crawler -p -e "DELETE FROM fanqie_ranks WHERE batch_date = '2026-03-14'"

# 清空缓存
python -m cli.main clear-cache
```

### Q: 如何查看 API 文档？

A: 启动服务后访问 `http://localhost:8000/docs` 查看 Swagger UI 文档。

### Q: 如何添加新的爬虫站点？

A: 参考 [架构文档](architecture.md) 中的"扩展新站点"章节。
