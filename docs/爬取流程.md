# 番茄小说爬取流程

## 概述

本文档详细描述番茄小说爬虫的核心流程、模块架构和反爬策略。

## 核心模块架构

```
┌─────────────────────────────────────────────────────────────┐
│                    FanqieSpider (主爬虫)                     │
│  协调各组件：BrowserManager / HumanSimulator /               │
│           PageParser / DataProcessor                         │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌──────────────┐    ┌──────────────────┐   ┌──────────────┐
│ BrowserManager│    │ HumanSimulator   │   │ PageParser   │
│ 浏览器管理    │    │ 人类行为模拟     │   │ 页面解析     │
└──────────────┘    └──────────────────┘   └──────────────┘
        │                     │                     │
        └─────────────────────┼─────────────────────┘
                              ▼
                    ┌──────────────────┐
                    │ DataProcessor    │
                    │ 数据处理 + 缓存   │
                    └──────────────────┘
                              │
                              ▼
```

## 爬取流程

### 1. 初始化阶段

```python
# FanqieSpider.__init__()
- 初始化 SpiderConfig (配置)
- 初始化 FontMapper (字体解码器)
- 初始化 BrowserManager (浏览器管理)
- 初始化 HumanSimulator (人类行为模拟)
- 初始化 PageParser (页面解析器)
- 初始化 DataProcessor (数据处理器)
- 获取当前批次日期 (UTC+8)
```

### 2. 主流程 (run 方法)

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 从数据库加载昨日数据到 Redis 缓存                              │
│ 2. 初始化浏览器 (init_browser)                                 │
│ 3. 爬取榜单首页 (crawl_rank_home)                              │
│    └── 获取所有分类列表 (如：男频阅读榜 - 玄幻)                    │
│                                                               │
│ 4. 遍历每个分类 (crawl_category_books)                         │
│    ├── 访问分类榜单页                                         │
│    ├── 快速滚动到底部加载所有书籍                              │
│    ├── 解析书籍项 (parse_book_item)                           │
│    │   ├── 排名、书名、作者 (字体解码)                         │
│    │   ├── 在读人数 (含"万"/"亿"单位转换)                      │
│    │   ├── 简介、封面 URL                                     │
│    │   └── 榜单更新时间                                       │
│    └── 批量保存基础数据到数据库                                │
│                                                               │
│ 5. 爬取书籍详情 (可选，crawl_detail=True)                       │
│    ├── 数据处理器预筛选 (process_batch_books)                  │
│    │   ├── 检查 Redis 缓存                                      │
│    │   ├── 检查数据库历史数据                                  │
│    │   └── 跳过已爬取/已完结的书籍                             │
│    │                                                          │
│    ├── 逐本爬取详情 (crawl_book_detail)                        │
│    │   ├── 先访问首页建立信任                                  │
│    │   ├── 访问书籍详情页                                     │
│    │   ├── 检查页面有效性                                     │
│    │   │   ├── 检查 URL 是否被重定向                            │
│    │   │   ├── 检查是否为空白页 (被反爬封锁)                    │
│    │   │   └── 失败时刷新浏览器指纹 + 重试 (最多 3 次)            │
│    │   ├── 人类行为模拟                                       │
│    │   │   ├── 缓慢滚动到底部加载章节                          │
│    │   │   └── 鼠标移动浏览章节                               │
│    │   ├── 解析详情页 (parse_book_detail)                      │
│    │   │   ├── 书籍状态 (连载/完结)                            │
│    │   │   ├── 章节列表                                       │
│    │   │   └── 最后更新时间                                   │
│    │   └── 保存数据 + 设置 Redis 缓存                            │
│    │                                                          │
│    └── 爬取失败降级方案                                       │
│        └── 从数据库查询历史数据复用                            │
│                                                               │
│ 6. 完成统计                                                   │
│    └── 输出：基础数据量、详情更新数、跳过数、缓存命中数、复用数  │
└─────────────────────────────────────────────────────────────┘
```

## 核心组件职责

### 1. BrowserManager (浏览器管理)

**文件**: `browser_manager.py`

**核心功能**:

```python
# 初始化浏览器
init_browser()
├── 启动 Playwright Chromium
├── 随机 User-Agent
├── 随机视口尺寸 (1920x1080, 1366x768, etc.)
├── 随机设备缩放比 (1, 1.25, 1.5, 2)
├── 注入隐藏自动化特征的 JS
│   ├── 隐藏 navigator.webdriver
│   ├── 伪装 plugins/languages
│   ├── 伪装硬件信息 (hardwareConcurrency, deviceMemory)
│   ├── 伪装 chrome 对象
│   ├── 伪装 permissions
│   └── 随机化 Canvas/WebGL 指纹
└── 设置 extra_http_headers

# 刷新浏览器指纹 (遇到封锁时)
refresh_fingerprint()
├── 创建新的 BrowserContext
├── 随机化硬件指纹
├── 随机化 Canvas/WebGL 指纹
└── 保留 Cookie 和登录状态
```

### 2. HumanSimulator (人类行为模拟)

**文件**: `human_simulator.py`

**核心功能**:

```python
# 模拟人类阅读
simulate_human_reading()
├── 随机停顿 0.5-1 秒
├── 随机滚动 1-2 次 (每次 200-600px)
├── 返回顶部
└── 再次向下滚动

# 模拟人类滚动到底部
human_scroll_to_bottom()
├── 缓慢滚动到底部
├── 延迟 2-3 秒 (等待 JS 加载)
└── 重复 2 次确保加载完成

# 模拟鼠标浏览章节
simulate_mouse_browse()
├── 在章节列表区域随机移动
├── 随机停留 0.1-0.3 秒
└── 滚动回顶部

# 快速滚动 (用于非详情模式)
quick_scroll_to_bottom()
└── 快速滚动到底部，延迟 0.2 秒
```

### 3. PageParser (页面解析)

**文件**: `page_parser.py`

**核心功能**:

```python
# 解析榜单首页
parse_rank_categories(html)
└── 提取分类链接 (gender_id, board_type, cat_id, cat_name)

# 解析榜单页书籍项
parse_book_item(item, category, board_name, batch_date)
├── 排名 (从 .book-item-index 提取)
├── 书名 (字体解码)
├── 作者 (字体解码)
├── 指标数据 (在读人数/热力值)
│   └── 解析 "12.5 万" → 125000
├── 简介 (从 .desc.abstract 提取，字体解码)
├── 封面 URL
├── 书籍状态 (连载/完结)
└── 榜单更新时间

# 解析书籍详情页
parse_book_detail(html)
├── 书籍状态 (连载/完结)
├── 章节列表 (从 .chapter-list 提取)
└── 最后更新时间
```

### 4. DataProcessor (数据处理)

**文件**: `data_processor.py`

**核心功能**:

```python
# 批量预筛选 (判断哪些书需要爬取)
process_batch_books(books, batch_date)
├── 检查 Redis 缓存
│   ├── 已完结 → 从数据库复制历史章节
│   └── 连载中 + 缓存时间 >= 榜单更新时间 → 跳过
├── 检查数据库历史数据
│   └── 已完结 → 从数据库复制历史章节
└── 返回需要爬取的书籍列表

# 保存基础数据
save_books_to_db(books, batch_date)
└── 批量插入 fanqie_ranks 表

# 保存书籍详情
save_book_detail(book_id, batch_date, book_status, chapter_list_json)
├── 更新数据库
└── 设置 Redis 缓存

# 复用历史章节
reuse_chapters_from_history(book_id, batch_date)
└── 从历史批次复制章节数据
```

## 反爬规避策略

| 策略 | 实现方式 | 位置 |
|------|----------|------|
| **隐藏自动化特征** | 注入 JS 隐藏 `navigator.webdriver`，伪装 plugins/languages | BrowserManager |
| **浏览器指纹随机化** | 随机 UA、视口、设备缩放比、硬件信息 | BrowserManager |
| **人类行为模拟** | 随机延迟、滚动、鼠标移动 | HumanSimulator |
| **完整请求头** | 完整的 `Accept-*`、`Sec-Fetch-*` 请求头 | SpiderConfig |
| **失败重试 + 刷新指纹** | 检测空白页时刷新指纹后重试 (最多 3 次) | FanqieSpider |
| **先访问首页** | 访问详情页前先访问首页建立信任 | FanqieSpider |
| **缓慢滚动加载** | 模拟人类缓慢滚动，等待 JS 加载章节 | HumanSimulator |

## 字体解码机制

番茄小说使用自定义字体文件加密数字和常用字：

```
原始页面文本 → OCR 映射表解码 → 真实文本

例如:
- 页面显示："𤭢者 12 万" (私有区 Unicode 字符)
- 解码后："作者 12 万"
```

**解码流程** (`pipeline/font_mapper.py`):

1. 加载 OCR 映射表 (`ocr_mapping.json`)
2. 遍历文本中的每个字符
3. 如果是私有区字符 (Unicode 0xE000-0xF8FF)，查表映射
4. 返回解码后的文本

**解码率**: 99%+

## 数据缓存策略

### Redis 缓存结构

```json
{
  "book:{book_id}": {
    "book_status": "已完结",
    "last_crawl_time": "2026-03-14 10:30:00"
  }
}
```

### 判断逻辑

```
1. 已完结书籍
   └── 直接从数据库复制历史章节 (不爬取)

2. 连载中 + 缓存时间 >= 榜单更新时间
   └── 跳过 (不爬取)

3. 其他情况
   └── 需要爬取详情页
```

### 缓存命中流程

```python
check_book_needs_crawl(book, batch_date)
├── 检查 Redis 缓存
│   ├── 已完结 → copy_chapters_from_history_by_status()
│   └── 连载中 + 未更新 → reuse_chapters_if_unchanged()
│
├── 检查数据库
│   └── 已完结 → copy_chapters_from_history_by_status()
│
└── 返回 (needs_crawl, reason)
```

## 补充爬取 (refill)

针对缺少章节数据的书籍的补充爬取流程：

```
1. 查询当天批次中 chapter_list_json 为空的书籍 ID
   └── get_books_without_chapters(batch_date)

2. 尝试从历史批次复用章节数据
   └── copy_chapters_from_history(book_id, batch_date)

3. 对仍需爬取的书籍，逐个访问详情页获取章节
   └── crawl_book_detail(book)

4. 批量更新数据库
   └── update_detail_batch(updates, batch_date)
```

## 使用示例

### 基础爬取

```bash
# 爬取所有分类，每个分类 30 本书，不爬取详情
python -m cli.main crawl --all --limit 30

# 爬取指定分类
python -m cli.main crawl --idx 0 --limit 30
```

### 爬取详情

```bash
# 爬取所有分类 + 详情 (章节列表)
python -m cli.main crawl --all --limit 30 --detail
```

### 补充爬取

```bash
# 补充爬取缺失章节的书籍
python -m cli.main refill --limit 100
```
